#### **Day 1：数组 & 字符串**
- **重点技巧**：双指针、滑动窗口、哈希优化
- **必做题目**：
    - 两数之和（哈希表）
    - 最长无重复子串（滑动窗口）
    - 接雨水（双指针/动态规划）
- 已做题目：
	1. 合并两个有序数组：为什么**倒序双指针**一定能**避免覆盖**
	2. 移除元素：把符合条件的元素放到0，1，2...的位置上
	3. 删除有序数组中的重复项：**反向判定**；库函数：slices.Compact(nums)
	4. 删除有序数组中的重复项2️⃣：最多出现2次重复，本质还是快慢指针
	5. 多数元素
	6. 轮转数组：**反转再反转**
	7. 买卖股票的最佳时机1️⃣：维护买入价格最小值和利润最大值；
	8. ~~买卖股票的最佳时机2️⃣：（动态规划）~~
	9. 跳跃游戏1️⃣：（能否跳到）**维护最右可达位置mx**；合并区间：每个 numsi​ 看成闭区间 [i,i+numsi​]，问题变成判定这 n 个区间能否合并成一个大区间
	10. 跳跃游戏2️⃣：（最小跳数）建桥法理解；遍历到n-2。**记忆化搜索**（在遍历过程中维护下一次最远可达的位置）
	11. H指数：**计数排序**，再统计大于h的论文数量
	12. 合并区间：左端点从小到大排序。。
	13. 除自身以外数组的乘积：前后缀数组
	14. 加油站：折线图理解；**总油量检查**
	15. 罗马数字转整数：对特殊情况的归一化处理
- **关联后端场景**：URL参数解析（字符串处理）、API限流（滑动窗口计数器）

### **时间利用技巧**
2. **碎片时间利用**：
    - 通勤/排队时用手机APP（如LeetCode Mobile）看题目描述和思路。
    - 午休时手写伪代码，晚上回家再调试。
3. **优先级策略**：
    - 先掌握 **代码模板**（如反转链表、DFS），再优化细节。
    - 每类题型至少刷2题，1题中等+1题困难（优先选高频题）。
4. **调试优化**：
    - 遇到卡壳时，先写注释再补代码（体现思路）。
    - 对比自己的解法和题解，分析复杂度差异。
---
### **注意事项**
5. **避免完美主义**：不必AC所有题目，重点是理解套路。
6. **代码规范**：面试官会看变量命名和边界处理（如判空、索引越界）。
7. **心态调整**：遇到难题时联想实际场景（如LRU缓存就是HashMap+链表）。
