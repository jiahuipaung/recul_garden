## 订单服务相关
1. **为什么选择 MongoDB 存储订单信息？** MongoDB 是一个文档型 NoSQL 数据库，适合处理订单这种结构灵活且数据量大的业务场景。选择 MongoDB 的原因主要是：
	1.  **高性能**：订单数据通常会有大量的读写操作，MongoDB 提供了高效的查询和写入性能，尤其在单文档查询时表现优秀。
	2. **灵活的 schema 设计**：订单数据可能存在不同的字段和格式，MongoDB 的文档结构使得我们能根据需求灵活调整数据模型，而不需要做复杂的表结构迁移。
	3. **水平扩展性**：随着业务的增长，MongoDB 支持横向扩展（Sharding），适应大规模订单数据的存储需求。
