Cilium用于**透明地保护**使用Docker 或 Kubernetes部署的应用服务之间的**网络连接**。
Cilium 的基础是一项名为 eBPF 的新 Linux 内核技术，该技术能够在 Linux 本身内动态插入强大的安全可见性和控制逻辑。由于 eBPF 在 Linux 内核中运行，因此无需更改应用程序代码或容器配置即可应用和更新 Cilium 安全策略。

eBPF 能够以前所未有的精细度和效率实现对系统和应用程序的可见性和控制。

# cilium
### 1. **微服务架构中的挑战与解决方案**
- **动态环境的挑战**：
    - 微服务中容器的生命周期短暂，IP 地址频繁变动。
    - 传统基于 IP 和端口的安全模型难以扩展，特别是在高动态性环境中。
    - 应用层协议端口（如 HTTP 的 TCP 80）无法区分具体服务流量。
- **Cilium 的解决方案**：
    - 利用 eBPF，以服务、Pod 或容器的身份作为标识，提供第 3 层到第 7 层的安全隔离。
    - 在应用层（如 HTTP）进行精细过滤，支持动态调整的安全策略。
    - 提供更强的扩展性，即使在数十万规则的情况下也能高效运行。
---
### 2. **功能概述**
#### (1) **API 安全保护**
- 支持现代协议（REST/HTTP、gRPC、Kafka 等）过滤。
- 示例：
    - 允许特定路径的 HTTP 请求（如 `GET /public/.*`），拒绝其他请求。
    - 为 Kafka 消息过滤指定主题生产者和消费者权限。
    - 在 HTTP REST 调用中强制验证特定的请求头格式。
#### (2) **基于身份的安全通信**
- 给共享同一安全策略的容器分配安全身份。
- 身份与数据包绑定，允许接收节点验证身份，无需频繁更新防火墙规则。
#### (3) **外部服务访问保护**
- 支持基于 CIDR 的入口与出口访问控制，确保对外部服务的安全限制。
---
### 3. **网络模型**
- **简单的第 3 层网络**：
    - 支持跨集群的扁平网络连接。
    - 基于主机范围的 IP 分配，无需复杂的协调。
- **多节点模型支持**：
    - **覆盖网络**：通过 VXLAN、Geneve 等封装实现，适用于大多数环境。
    - **本机路由**：使用 Linux 原生路由表，适合高级用户和原生 IPv6 网络。
---
### 4. **负载均衡**
- **南北向负载均衡**（外部流量）：
    - 优化的 eBPF 实现，支持直接服务器返回（DSR）和一致性哈希。
- **东西向负载均衡**（内部流量）：
    - 在 TCP 连接建立时高效完成服务到后端的流量转换，减少 NAT 开销。
---
### 5. **带宽管理**
- 通过 eBPF 和 EDT（Earliest Departure Time）实现高效流量限制，减少尾部延迟并优化多队列网络接口性能。
---
### 6. **可观测性与故障排除**
- 提供丰富的事件元数据，可视化网络流量。
- 支持通过 Prometheus 导出关键指标，方便仪表板集成。
- 集成 **Hubble**：
    - 服务依赖关系图。
    - 操作监控与警报。
    - 基于流日志的应用与安全可见性。
