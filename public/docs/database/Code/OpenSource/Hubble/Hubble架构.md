可以解决以下问题：
1. **服务依赖关系和通信图**
	- 哪些服务正在相互通信？通信频率如何？服务依赖关系图是什么样的？
	- 正在进行哪些 HTTP 调用？服务从哪些 Kafka 主题消费或生成哪些内容？
2. **网络监控和警报**
	- 是否有任何网络通信失败？通信失败的原因是什么？是 DNS 问题吗？是应用程序问题还是网络问题？通信在第 4 层 (TCP) 还是第 7 层 (HTTP) 上中断？
	- 过去 5 分钟内哪些服务遇到过 DNS 解析问题？哪些服务最近遇到过 TCP 连接中断或连接超时？未答复的 TCP SYN 请求的比率是多少？
3. **应用程序监控**
	- 特定服务或所有集群的 5xx 或 4xx HTTP 响应代码的发生率是多少？
	- 我的集群中 HTTP 请求和响应之间的第 95 和第 99 百分位延迟是多少？哪些服务表现最差？两个服务之间的延迟是多少？
4. **安全可观察性**
	- 由于网络策略，哪些服务的连接被阻止？哪些服务已从集群外部访问？哪些服务已解析特定的 DNS 名称？
